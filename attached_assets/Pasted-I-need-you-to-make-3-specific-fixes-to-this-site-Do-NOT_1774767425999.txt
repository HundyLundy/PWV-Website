FIX 1 — Add a vercel.json file to fix all 404 routing errors

The site is a React SPA (single-page app). When Vercel serves a direct URL like /colorado, /case-studies, /proposal, or /texas, it looks for a physical file at that path and returns 404 because there is no file — the routing is handled by React in the browser. The fix is a vercel.json config file that tells Vercel to always serve index.html for any route.

Create a new file at the root of the project called vercel.json with exactly this content:

JSON


{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}



That's it for Fix 1. This will fix Colorado, Texas, Case Studies, Get a Proposal, and every other route that currently 404s.
