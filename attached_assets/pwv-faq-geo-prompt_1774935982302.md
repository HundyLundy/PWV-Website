## THE RULE — Apply to Every Page

**Rule 1 — Direct-Answer Opening:**
Every page's hero subheadline or first body paragraph must answer the page's core question in 1–2 sentences, with a specific number. The answer must appear before any marketing copy. AI systems scan the first paragraph first — if the answer is buried, we don't get cited.

**Rule 2 — Unique FAQ Section:**
Every page gets a visible FAQ section near the bottom (above the CTA, below the main content). Each FAQ must be unique to that page's topic — no copy-pasting generic questions across pages. Minimum 4 questions per page, maximum 6.

**Rule 3 — JSON-LD FAQ Schema:**
Every page that has an FAQ section must also have a `<script type="application/ld+json">` FAQ schema block injected into the page's `<head>`. The schema must exactly match the visible FAQ questions and answers on the page.

## TECHNICAL IMPLEMENTATION

For every page that receives a FAQ section, inject the following JSON-LD schema block into the page's `<head>`. The `@type`, `mainEntity` array must exactly match the visible FAQ questions and answers:

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "[Question text exactly as shown on page]",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "[Answer text exactly as shown on page]"
      }
    }
  ]
}
```

If a page already has other JSON-LD schema (LocalBusiness, Service, Article), add the FAQPage schema as a second separate `<script type="application/ld+json">` block — do not merge them into one object.
